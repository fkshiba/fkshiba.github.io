Modernizando RIBs

Uma Nova Era para a Arquitetura Android

O Ponto de Partida: Desafios

❌ Android Views (XML)

  • Lógica e layout separados
  • Bugs de estado (state bugs)
  • Dificuldade para testar

❌ RxJava

  • Curva de aprendizado alta
  • Debugging complexo ("stacktrace hell")
  • Overhead de performance

A Solução: Três Pilares da Modernização

Kotlin Coroutines

Async simples e leve.

Jetpack Compose

UI declarativa em Kotlin.

Padrão MVI

Fluxo de dados previsível.

Coroutines vs. RxJava

RxJava

fun getUser(id: String): Single<User> {
    // ...
}

// No Interactor
getUser("123")
    .subscribeOn(Schedulers.io())
    .observeOn(AndroidSchedulers.mainThread())
    .subscribe({ user ->
        // update UI
    }, { error ->
        // handle error
    })

Kotlin Coroutines

suspend fun getUser(id: String): User {
    // ...
}

// No Interactor (dentro de um CoroutineScope)
lifecycleScope.launch {
    try {
        val user = getUser("123")
        // update UI
    } catch (e: Exception) {
        // handle error
    }
}

Jetpack Compose vs. XML

XML View

<Button
    android:id="@+id/my_button"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:text="Clique aqui" />
// Activity/Fragment
val button = findViewById<Button>(R.id.my_button)
button.setOnClickListener { 
    viewModel.onButtonClicked()
}

Jetpack Compose

@Composable
fun MyScreen(viewModel: MyViewModel) {
    val state by viewModel.uiState.collectAsState()

    Button(onClick = { viewModel.onButtonClicked() }) {
        Text("Clique aqui")
    }
}

  private val presenter: MyPresenter

  fun onEvent() {
    // ...

class MyInteractor : PresenterlessInteractor<
    MyState, MyEvent, MyEffect>(...) {
    
    override fun handleEvent(event: MyEvent) {
        // ...
        setState { copy(isLoading = true) }
        // ...
    }
}

O Fluxo de Dados MVI

Benefícios da Nova Arquitetura

  • ✅ Velocidade: Desenvolvimento de UI mais rápido.
  • ✅ Performance: Menos consumo de memória.
  • ✅ Confiabilidade: Menos bugs de estado.
  • ✅ Testes: Lógica de negócio isolada e fácil de testar.
  • ✅ Coesão: UI e lógica na mesma linguagem (Kotlin).
  • ✅ À Prova de Futuro: Alinhado com as práticas modernas do Android.

Rollout e Migração

Uma Abordagem Evolutiva

  • Adoção Opt-In: Sem migração forçada.
  • Rollout em Fases: Começar com novas features.
  • Interoperabilidade: O novo e o antigo coexistem.

Conclusão

Modernizar os RIBs é investir em um futuro mais produtivo, performático e agradável para o desenvolvimento Android.
